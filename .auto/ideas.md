# Deferred ideas

- Add adaptive large-diff sharding that balances files by changed-line weight while preserving a cross-file manifest for every specialist.
- Add a read-only context-gathering extension tool that fetches PR metadata, diff, identity, duplicate markers, convention paths, and baseline discovery concurrently in one orchestrator turn.
- Reuse a single secure temporary system-prompt directory across a batch instead of creating/removing one per pass; likely a small startup win.
- Profile child Pi startup and global extension discovery; consider explicitly disabling unrelated child extensions if tool/model behavior remains identical.
- Add a diverse offline corpus of synthetic diffs with seeded cross-file, security, correctness, and nit findings for repeatable semantic quality comparisons alongside periodic real-model reviews.
- Revisit one-wave context/validation scheduling with repeated samples: it reduced initial gathering from two turns to one and validation from five turns to two, but its first real-model sample was 47 seconds slower overall because batch/model variance dominated (445s batch vs 410s best).
- Combine parent validation batching with specialist evidence batching: specialist batching cut the six-pass batch from 410s to 324s and increased retained findings from 9 to 10, but four parent validation waves raised outside-batch time to 494s and total to 818s. The combination later succeeded at 632s total with all gates passing.
- Consider adaptive eight-pass fan-out only for unusually large/high-risk PRs. On the 35 KB benchmark it found 11 issues (vs six-pass best 8) but took 702s: batch stayed near best at 260s while the larger candidate set raised parent validation/synthesis to 442s.
- Same-model transient `Model not found` recovery preserved all ordinary-review gates but did not activate or improve a healthy benchmark. Revisit only with deterministic failure injection or as a separate reliability goal; never substitute a weaker model.
- Evaluate a runner/API path with true provider-managed caching for the unchanged full-diff prefix shared by heavy passes. The installed Pi CLI exposes sessions but no explicit prompt-cache or connection-reuse control; do not simulate it by weakening context or reusing contaminated review sessions.
- Adaptive lead-plus-verifier trial: one lead plus three verifier groups completed all structural gates and returned a P1/P2, but took 803.7s versus retained fixed-mode samples. Prompt-owned staged orchestration serializes lead discovery and verification; do not retain without corpus recall evidence and substantial latency gains.
- Discovery-only heavy specialists trial: all five discovery-mode passes and relevance gates passed, but total was 745.6s (454.9s heavy tail, 134.0s parent validation). Moving definitive substantiation to the parent did not shorten the critical specialist and increased parent work; rejected.
- Child phase trace after d4632d7: heavy first assistant output was 24-35s, actual tool execution only 0.06-1.8s, and non-tool/model time 194-515s. Tool I/O is negligible; the tail is additional model reasoning/finalization. Next test should eliminate specialist tool rounds while preserving parent validation, not optimize local reads.
- Tool-free one-shot heavy discovery trial: all five passes/relevance gates passed but total was 767.6s, heavy tail 550.5s, parent validation 98.2s/13 calls. Every heavy pass had toolElapsedMs=0, yet firstAssistantMs was 182-548s and essentially equaled completion. The model shifted reasoning into the single response; eliminating tool rounds did not reduce the tail. Rejected.
- Paired-heavy-lens trial: overview plus lifecycle/contracts and security/resources passed all three gates and found two relevant issues, but total was 670.1s, heavy tail 471.0s, and parent validation 85.5s. It did not beat the retained 452.0s balanced result and weakens independent lens isolation; configured tool work was only 65-167ms. Rejected.
- Direct extension-owned PR context/first-turn balanced dispatch: all five passes and relevance gates passed, with 525.7s total, 320.0s heavy tail, 110.5s parent work, and three findings. Implied pre-batch setup was still ~95.1s versus ~79s in the comparable instrumented balanced trace, so the lower total came from provider-tail variance rather than eliminating a parent turn. Added ~200 lines and no phase win; cleanup succeeded. Rejected.
